\section*{Reproducibility and data availability}
\label{sec:reproducibility}

\paragraph{Code and data availability.}
The complete source code and post-processing scripts used in this work are hosted on GitHub at \url{https://github.com/petr-henys/Smoothly-Coupled-Remodeling-with-Nonlocal-Stimulus-and-Fabric-Guided-Anisotropy.git} and permanently archived on Zenodo at \url{https://doi.org/10.5281/zenodo.18242655}.
The Zenodo record corresponds to the exact version used for this manuscript (release tag and commit hash recorded in the archive metadata) and includes (i) all parameter files, (ii) the anatomy- and gait-related inputs required for the femur case study, and (iii) the raw simulation outputs from which the figures were generated.
The repository is organized so that every simulation run is self-describing: each run stores a complete configuration snapshot, logs, and machine-readable metrics in the output directory together with the field data.

\paragraph{Software stack and execution.}
\rev{The solver is implemented in Python~3.10 on top of DOLFINx~0.10.0 (the FEniCSx finite element library), with PETSc~3.22 for distributed linear algebra, mpi4py for inter-process communication, and Basix~0.10.0 for finite element tabulation.}
The solver is designed for distributed-memory parallel execution and writes field output in a scalable parallel format suitable for large meshes and restart.
A minimal end-to-end example is the box benchmark, which runs out of the box and produces a self-contained results directory.
All parameter sweeps and figure-generation pipelines used in the manuscript are provided in the repository.

\paragraph{Modular staggered architecture and extensibility.}
The coupled problem is advanced by a partitioned block Gauss--Seidel scheme (mechanics $\rightarrow$ fabric $\rightarrow$ stimulus $\rightarrow$ density; \S\ref{sec:fp-aa}).
Each subproblem is encapsulated as an independent \emph{coupling block} exposing its state and output fields; the fixed-point solver, time integrator, I/O, and diagnostics operate on a registry of these blocks and therefore remain agnostic to the specific set of fields.
Adding a new field to the staggered scheme requires only (i) defining its state variable (and its previous-time counterpart), (ii) implementing one additional block that advances it, and (iii) registering the block in the desired Gauss--Seidel order.

\paragraph{Parallel execution and scaling.}
All simulations are MPI-parallel and require no shared-memory assumptions.
For parameter studies we additionally support concurrent execution of multiple independent runs by splitting the available MPI ranks into subcommunicators, enabling both strong scaling (single large run) and throughput scaling (many runs in parallel).
To document parallel performance, we include a strong scaling benchmark on the stiff femur case study.
We measure the wall-clock time per \emph{global} time step of the fully coupled multiphysics solve, including all four staggered blocks (mechanics, fabric, stimulus, density) and their fixed-point coupling iterations.
On this representative mesh ($N_{\mathrm{cells}}\approx 75\mathrm{k}$; see caption for DOF breakdown) the solver shows a slight super-linear speedup from 1 to 2 MPI ranks (likely due to cache effects) and maintains 56\% parallel efficiency at 8 ranks (speedup $4.45\times$), despite the modest problem size per rank (\cref{fig:strong_scaling}).

\paragraph{Verification.}
Core infrastructure (time integration, MPI-consistent diagnostics, I/O) is covered by automated regression tests (serial or under MPI).
